# 一、历史背景

时间：1950 年代，美国 IBM 实验室。

作者：Arthur Samuel（图灵奖获得者，AI 先驱）。



核心问题：能否让计算机不靠人编码规则细化，而是通过经验提升水平？作者把这称为“机器学习”（Samuel 是最早给出该术语操作性定义的人之一）。



选择跳棋（Checkers）的理由：

规则简单、状态有限但庞大（适合做“搜索+学习”的试验台）。

当时机器算力极其有限，需要在启发式与自适应之间取得平衡。

目标：构造一个程序，能通过对弈（包括自我对弈）持续改进棋力，并给出可重复的实验与量化的结果。

# 六、自我对弈与训练流程（Self-Play Pipeline）

循环：生成对局 → 记录关键中间局面与估值 → 对照更深搜索/终局结果 → 回写到权重与记忆库。

随机性：在等值走法间注入少量随机，避免陷入重复轨迹（探索 vs 利用）。

版控：定期对比“当前模型”与“旧版本”对战胜率，用胜场验证进步（A/B 评价）。

持续学习：记忆库扩张 + 权重缓慢收敛，棋力随时间单调提升为目标。

# 八、工程实现细节

局面签名/检索：为局面构造紧凑“指纹”，加速查找（开局库/置换表的前身思路）。

开局书（book learning）：把高质量开局路径写入库，开局阶段快速、稳健。

次序启发：优先展开“吃子/强制应手”等，使剪枝更有效。

终局/近终局表：小范围内可做穷举或深表，提供高置信度的训练信号。


（工程技巧减小了算力负担，然而在算力充足的情况下应该舍弃它们）

# 九、局限

算力不足：搜索深度受限，评价函数必须极简（牺牲精度）。

学习信号噪声：对局输赢与单一局面优劣并非一一对应，权重更新带噪（近似 TD 的早期尴尬）。

特征工程依赖：评估效果强依赖人手设计的特征；程序不会“自己找特征”（深度表征学习尚未出现）。

最优保证缺失：启发式+经验更新无法保证全局最优（工程取向大于理论最优）。


# 十、对ai的影响

1.机器学习的开创者：Samuel 通过跳棋实验系统性地提出了 机器学习 的框架。  
2.开创了自我对弈路线：从 Samuel →（多年后）TD-Gammon（Tesauro, 1992）→ AlphaGo/AlphaZero（DeepMind, 2016–2018），谱系清晰。  
3.哈希/置换表/开局库等实战型博弈程序的基础工程在这篇工作里已有雏形。  
4.评价函数 + 搜索成为自适应搜索算法的范式。
