# 一、技术报告总结

模型形态：GPT-4 被定位为大规模多模态模型，既能接收文本也能接收图像输入，输出文本（视觉能力的部署、安全评估另见 GPT-4V System Card）。

训练范式：延续“自回归 Transformer + 预测下一 token”的大一统预训练思路，之后通过对齐（RLHF 等）做后训练；但不公开参数规模、训练算力、数据配比等关键细节。

能力评测：在大量专业与学术基准上达到接近人类甚至通过型表现（例如统一司法考试 UBE 的高分段结论在报告中出现），同时强调仍存在幻觉与可靠性问题。

安全与部署：报告与 System Card 详细记录了红队测试、风险场景与缓解（生物、虚假信息、隐私、网络安全等）。

# 二、与 GPT-3 的核心差异

## 1.单模态 → 多模态：

GPT-3：仅文本输入。

GPT-4：原生支持图像+文本输入（论文中展示与系统卡分述视觉评测与风险）。→ 带来图像理解、图表读写、场景问答等新型任务能力与新风险治理面。

GPT-4：这幅图片展示了一个 "闪电电缆" 适配器的包装，分为三个面板。面板1：一部智能手机，插入了一个VGA连接器（一个大型的蓝色15针连接器，通常用于计算机显示器）的充电端口。面板2："闪电电缆" 适配器的包装，上面有一个VGA连接器的图片。面板3：VGA连接器的特写，其末端有一个小型闪电连接器（用于为iPhone和其他苹果设备充电）。这幅图的幽默之处在于把一个大型的过时VGA连接器插入一个小型的现代智能手机充电端口的荒谬性。

## 2.信息披露方式：开放细节 → 能力报告

GPT-3 论文公开了参数规模（175B）、上下文窗口（2048）、数据构成与采样比例（Common Crawl、WebText2、Books、Wikipedia 的配比与有效 epoch），并系统做了zero/one/few-shot三设定评测。

GPT-4 报告刻意不披露参数量、训练数据配方与算力，仅聚焦能力、局限与安全；强调多模态、红队与风险缓解流程。→ 学术可复现性与外部审计性降低，但部署安全信息更丰富。

## 3.安全框架与红队：初步讨论 → 系统化卡片与多模态风险库

GPT-3：在论文里讨论偏见/滥用，但重点仍在规模规律与 ICL（提示内学习）。

GPT-4：发布了System Card与GPT-4V(ision) System Card，详述红队过程、能力边界、敏感场景（如人脸、私人信息、危险活动）以及上线前后缓解策略。→ 标志着从“模型研究”走向“产品级风险管理”。

# 三、启示

1.提示与链式思维（CoT）仍重要：即便模型更强、上下文更长，结构化提示（角色+任务+格式+示例）与分步归纳，依然是控制输出稳定性的关键。（与 GPT-3 的 few-shot 方法论一脉相承）

2.把安全前置：在落地场景中，参考系统卡（system card）的敏感类别与缓解清单（内容过滤、拒答策略、最小必要信息、人工复核）做产品级合规设计。
